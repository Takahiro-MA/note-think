# note記事 半自動制作パイプライン — 統合設計（軽量版）

> ①売れる形のノウハウ（`01_paid_note_playbook.md` / `02_note_monetization_book_notes.md`）
> ＋ ②Kindle由来のワークフロー枠組み（`03_workflow_framework.md`）
> を統合し、**note向けにカジュアル最適化**した実装設計。2026-06-13 合意。

## 設計思想

- **全記事一律をやめる**。記事の"格"でプロセスの重さを変える（①の「無料で検証→有料に投資」と同じ論理）。
- ②の3思想のうち、**仕様書ファースト**は5行に圧縮、**多視点レビュー**は任意化、**段階ゲート**は人間1点に削減。
- ①は「note市場で売れる形」を担保する制約として、②のステップに**内蔵**する（別工程にしない）。

## 既定フロー（カジュアル・3ステップ・人間ゲート1点）

```
/article <ネタ>
  ├ ステップ0: ネタ採点（テーマ採点表75/60/59 ＋ 魂チェック=生々しい体験+感情）
  ├ ステップ1: 5行設計（core/reader/free_value/paid_value/price/tags/experience）→ フロントマター
  └ ステップ2: 初稿執筆（構成テンプレ・無料黄金順序・タイトル5型・区切り技を内蔵）
        ↓
/check-compliance <file>   ← 自動・人手ゼロ（規約20項目/景表法/著作権）
        ↓
/publish <file>            ← 唯一の人間ゲート（公開GO）→ note下書き投稿
        ↓
（note編集画面で 有料ライン・価格を人が設定）
```

任意: `/review <file>` … 有料の主力記事のときだけ多視点レビュー（編集者/読者/拡散の並列）。

## 格によるプロセス可変

| 記事の格 | フロー |
|---------|--------|
| 無料 / お試し | `/article`（簡略）→ 自分でサッと確認 → `/publish`。レビューなし |
| 有料・主力 | `/article` → `/review`（1回）→ 反映 → `/check-compliance` → `/publish` |

## ①ルールが②のどこに刺さっているか

| Phase | ②枠組み | 内蔵した①ルール |
|-------|--------|----------------|
| ネタ採点 | GO/NO-GO | テーマ採点表・HARM法則に縛られない「体験＋感情」判定 |
| 5行設計 | 仕様書ファースト(圧縮) | 無料/有料分割点・価格帯・タグ3-7個・一次体験(魂) |
| 執筆 | design準拠 | 構成と文量・無料エリア黄金順序・タイトル5型・目次は無料末尾・結論は無料で予告 |
| 公開前 | 人間承認② | **NG/不当表示の自動チェック（②に無い・①が足した必須ゲート）** |

## 実装ファイル

- `.claude/commands/article.md` — 設計＋執筆
- `.claude/commands/check-compliance.md` — 自動コンプラ検査
- `.claude/commands/review.md` — 任意・多視点レビュー
- `.claude/commands/publish.md` — チェック→人間承認→下書き投稿
- `article-status.md` — 記事の状態管理
- `articles/<slug>/article.md` — 記事本体（フロントマターに設計を内蔵）
- グループAの思考モード（`/brainstorm` `/sparring` `/evaluate` `/decide`）は既存のグローバルskillを流用（再実装しない）

## マネタイズ方針（2026-06-13 Takahiro判断）

- **アフィリエイトリンク（Amazon等）を含む記事は無料にする**。課金＋アフィ報酬の二重取りは読者に鼻につき、信頼を損なうため。マネタイズはアフィに一本化。
- 有料化は、外部収益の無い純粋なノウハウ/体験記事に限る。
- アフィを含むなら PR表示は必須（無料でもステマ規制対象）。

## 既知の制約 / TODO

- note APIは**下書き保存のみ**。有料記事の無料/有料の分割と価格設定は note編集画面で人が行う（`client.py` は paywall 未対応）。将来 `client.py` に有料設定APIを足せれば自動化余地あり。
- 有料ラインの目印は**見える1行マーカー**で本文に入れる（HTMLコメントは note 上で見えないので不可）。人が編集画面で設定後にその行を削除する。
- 一次体験（experience）は自動生成できない＝**Takahiroの入力が必須**。ここがパイプラインの律速かつ価値源泉。
