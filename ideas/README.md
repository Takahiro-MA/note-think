# ideas/ ストックの使い方（運用ルール）

このフォルダは note記事の「ネタのストック層」。新しい思いつきが、混乱なくここに溜まる仕組み。

## 新しい思いつきの落とし方（これだけ覚えればOK）
- **迷ったら `_netacho.md` の「🗑 未分類インボックス」に一行で放り込む。** 整える必要なし。
- あとは Noah が柱に振り分け、雑談で具体が溜まったら個別ファイルに昇格させる。

## パイプライン（ネタの一生）
💡 raw（一行） → 📝 draft（個別ファイル） → ✍ 執筆 → ✅ 投稿

| 状態 | 置き場所 |
|------|---------|
| 💡 raw | `_netacho.md` の柱 or 🗑未分類 に **一行**で |
| 📝 draft | `ideas/<slug>.md`（雑談で具体が溜まったら昇格・`status: idea-draft`）。`_netacho` からはリンクのみ |
| ✍ 執筆中 | `articles/<slug>/article.md` |
| ✅ 投稿 | `article-status.md`（note_id・URL・状態） |

## ファイルの役割（単一の真実源＝各事実は1箇所だけ）

| ファイル | 役割 |
|---------|------|
| `_netacho.md` | **地図／ハブ**。全ネタを状態つきで一覧＆リンク。詳細は持たない |
| `_self-inventory.md` | 人物像マスター（ChatGPT＋Claude棚卸し統合）＝主軸・声・素材の原料 |
| `_<theme>-series.md` | 柱/シリーズの tracker＋記事スレート（例: `_sobriety-series.md`） |
| `<slug>.md` | 個別ネタのドラフト（`status: idea-draft`） |
| `<theme>-log.md` 等 | 一次情報ソース（例: `sobriety-log.md`） |

## 命名・リンク規約
- `_` 接頭辞＝ハブ/メタ/横断ファイル。個別ネタは内容スラッグ（英小文字ケバブ）。
- リンクは `ideas/xxx.md` のパス（エディタでクリック可）。memory側は `[[name]]`。
- **重複を作らない**: 詳細は1ファイルに置き、他はリンクで指すだけ。

- 決定の経緯（日付つきハンドオフ） → `../notes/`
