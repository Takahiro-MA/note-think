`$ARGUMENTS` の記事を下書き投稿まで持っていく。**唯一の人間ゲート（公開GO）はここ。**

## 手順

1. **自動コンプラチェック**: `/check-compliance $ARGUMENTS` 相当を実行。FAIL があれば**ここで停止**し、修正を促す（投稿しない）。

2. **投稿前サマリ提示**: PASS なら以下を1画面で提示する:
   - タイトル / 想定文量 / type(free|paid) / 価格 / タグ
   - 無料・有料の分割点（`<!-- 有料ライン -->` の位置）
   - 冒頭リード3行
   - コンプラチェック結果（PASS）

3. **人間承認①（唯一のゲート）**: 「この内容で note に下書き投稿してよいですか？」とユーザに確認し、**承認を待つ**（マーカー `<<NEEDS-INPUT>>` を出して停止）。勝手に投稿しない。

4. **下書き投稿**: 承認を得たら、フロントマターを除いた本文で投稿する:
   ```bash
   cd /home/takahiroma/noah-workspace && python3 -m note_poster draft "<タイトル>" <本文ファイル>
   ```
   - ※ 実行パスの正典は `CLAUDE.md`「実行環境」節（上記はホスト用。コンテナ内は `/workspace/.venv/bin/python`）。
   - ※ フロントマターと `<!-- 有料ライン -->` は本文に含めない一時ファイルを作って投稿する。
   - ※ note APIは下書き保存のみ。**無料/有料の分割と価格設定は note編集画面で人が行う**（投稿後にURLを案内し、分割点の位置を伝える）。

5. **後処理**: 投稿成功なら `article-status.md` を更新（Phase=Published / URL記入）。下書きURLをユーザに案内し、「note編集画面で有料ラインと価格を設定してください」と添える。
